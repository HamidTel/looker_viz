Technical Documentation: Telecom Call Trace & Signaling Aggregation Pipeline

Executive Summary

This document outlines the architecture and logic of the Telecom Call Trace SQL pipeline. The pipeline ingests raw, multi-protocol control plane signaling data (SIP, GTP/ESM, NGAP, DIAMETER) and transforms it into a sequence-ready, topology-aware Call Detail Record (CDR) dataset.

The primary objective is to group individual network packets into unique call sessions, attribute events to specific network nodes (Caller vs. Called, UE vs. Core), calculate mobility and latency KPIs, and unpivot the data into a timeline format suitable for sequence diagrams and Gantt charts.

High-Level Conceptual Architecture

The pipeline is designed around four distinct logical layers, progressively transforming raw, stateless network packets into highly contextualized, sequence-ready call records.

1. Data Extraction & Normalization (base)

Purpose: Acts as the foundational ingestion layer. It filters raw network events for a specific time window, normalizes MSISDNs, resolves Radio Access Technology (RAT) fallbacks based on cell length, and bridges the calling address to the B-Party's IMSI via a reference table.

graph TD
    Start([Start: bq_control_plane]) --> Filter{Initial Filters}
    Filter --> |Time/Messages| MsgTypes[INVITE, BYE, NGAP MOD, GTP CBR, DIA ASR]
    MsgTypes --> Join[LEFT JOIN msisdn_ban]
    Join --> RAT_Calc[Determine RAT: 5G-NR, EUTRAN, UTRAN]
    Join --> PhoneFields[Normalize MSISDNs]
    Join --> TS_Logic[Extract Protocol Timestamps]
    RAT_Calc & PhoneFields & TS_Logic --> Output([Result: base])

2. Identity & Role Resolution (base_processed)

Purpose: Utilizes a 20-row preceding window to standardizes MSISDNs by prepending country codes and explicitly defines the A-Party (calling_imsi) and B-Party (called_imsi) based on SIP Client-Server interaction types.

graph TD
    Input([Input: base]) --> Window[Partition by subscriber_id]
    Window --> Concat1[Standardize Calling/Called Numbers]
    Window --> ID_Logic{Identify Role}
    ID_Logic --> |INVITE C-S| CallingIMSI[Assign calling_imsi]
    ID_Logic --> |INVITE S-C| CalledIMSI_A[Assign called_imsi]
    Concat1 & CallingIMSI & CalledIMSI_A --> Output([Result: base_processed])

3. Data Stitching (base_filled)

Purpose: Generates a pair_key_msisdn to link all messages between two specific numbers. It uses a lookback window function to propagate the identified caller/called IMSIs to mid-call signaling packets.

graph TD
    Input([Input: base_processed]) --> KeyGen[Create pair_key_msisdn]
    KeyGen --> Range[Lookback: 20 rows preceding]
    Range --> FillCalling[Fetch/Keep calling_imsi]
    Range --> FillCalled[Fetch/Keep called_imsi]
    FillCalling & FillCalled --> Output([Result: base_filled])

4. Sessionization & Directional Pivot (prep)

Purpose: The core processing engine. Identifies genuine new call attempts by looking for a >3s gap between INVITE packets, generating an MD5 hash. Pivots all multi-protocol events into Calling vs Called side columns.

graph TD
    Input([Input: base_filled]) --> Filter[Filter: Target IMSI]
    Filter --> LAG{Is INVITE and LAG > 3s?}
    LAG -->|Yes| NewCall[Generate call_id Hash]
    LAG -->|No| ExistingCall[call_id = NULL]
    Filter --> RoleSplit{Check Subscriber}
    RoleSplit -->|calling_imsi| C_Logic[Map to Calling Fields]
    RoleSplit -->|called_imsi| B_Logic[Map to Called Fields]
    NewCall & C_Logic & B_Logic --> Output([Result: prep])

5. Session ID Propagation (call_id)

Purpose: Executes a fill-down operation to propagate the specific call_id to all subsequent signaling packets belonging to that session.

graph TD
    Input([Input: prep]) --> Frame[Frame: UNBOUNDED PRECEDING to CURRENT ROW]
    Frame --> CheckID{Is call_id NULL?}
    CheckID -->|No| Keep[Use existing call_id]
    CheckID -->|Yes| Fetch[LAST_VALUE call_id IGNORE NULLS]
    Keep & Fetch --> Output([Result: call_id])

6. State Accumulation (summary)

Purpose: An aggregation layer that uses FIRST_VALUE and LAST_VALUE across an unbounded window to ensure every row contains the full state of the call. Calculates distinct RAT and Cell IDs for handovers.

graph TD
    Input([Input: call_id]) --> Part[Partition by last_call_id]
    Part --> SIP[First/Last INVITE & BYE]
    Part --> Bearer[First/Last Bearer State]
    Part --> Radio[Radio Mod / Abort State]
    Part --> DCount[Count unique cell_id & rat_name]
    SIP & Bearer & Radio & DCount --> Output([Result: summary])

7. KPI Calculation (metrics)

Purpose: Computes diagnostic timings (EPSFB latency, setup time, sync delays) and creates Mobility Path fingerprints.

graph TD
    Input([Input: summary]) --> Filter[Filter: BYE_T]
    Filter --> CBR_Check{Failure TS > Success TS?}
    CBR_Check --> CBR_Fail[Flag Failure + Error]
    CBR_Check --> KPIs[Calculate Latencies]
    KPIs --> Paths[Create Initial/Final RAT & Cell Paths]
    CBR_Fail & Paths --> Output([Result: metrics])

8. Classification & Unnesting (unpivoted_events)

Purpose: Finalizes call technology (VoLTE vs VoNR vs EPSFB) and transitions the dataset to a Long format using an UNNEST array for timeline generation.

graph TD
    Input([Input: metrics]) --> RAT_Check{Check RAT & Bearer}
    RAT_Check --> Classify[Label: VoNR, VoLTE, EPSFB]
    Classify --> UNNEST[UNNEST Array of STRUCTs into Event Rows]
    UNNEST --> Output([Result: unpivoted_events])

9. Topology & Timeline Assembly (Final Output)

Purpose: The presentation layer. Resolves hanging labels, calculates relative timeline, and maps abstract events to physical network topology (e.g., UE1, P-CSCF, MME, gNB).

graph TD
    Input([Input: unpivoted_events]) --> Elapsed[Calculate Relative Time: TS - Call Start]
    Elapsed --> NodeMap{CASE: event_name & Entry Type}
    NodeMap -->|BYE Client-Server| UE_Hangup[Source: UE1/UE2]
    NodeMap -->|BYE Server-Client| Net_Hangup[Source: P-CSCF]
    NodeMap -->|Bearer| Nodes[SGW ➔ MME]
    UE_Hangup & Net_Hangup & Nodes --> FinalOut([Result: Trace-Ready Dataset])
